# Конвертер единиц — ТЗ

> Саня (продакт) | 2026-03-29

## Цель продукта

Быстрый, интуитивный конвертер единиц измерения. Целевая оценка: 8.5+/10. Ключевой дифференциатор от конкурентов — скорость: пользователь получает результат за 1-2 действия, без лишних кликов.

Предыдущие продукты песочницы: калькулятор (7.9), таймер (8.2). Основные точки роста: polish взаимодействия, тактильная обратная связь, zero-friction UX.

---

## User Stories

1. Как пользователь, я хочу **ввести число и мгновенно видеть результат конвертации**, чтобы не тратить время на нажатие кнопки "Конвертировать".
2. Как пользователь, я хочу **поменять направление конвертации одним тапом** (swap), чтобы быстро посчитать обратно.
3. Как пользователь, я хочу **выбрать категорию (длина, вес, температура, объём)**, чтобы видеть только релевантные единицы.
4. Как пользователь, я хочу **видеть результат сразу во всех единицах категории**, чтобы не переключаться между парами.
5. Как пользователь, я хочу **скопировать результат в буфер обмена одним тапом**, чтобы вставить его в другое приложение.
6. Как пользователь, я хочу **видеть свои последние конвертации**, чтобы быстро вернуться к ним.

## Acceptance Criteria

### Функциональность

- [ ] 4 категории: длина, вес, температура, объём
- [ ] Длина: мм, см, м, км, дюймы, футы, ярды, мили
- [ ] Вес: мг, г, кг, фунты, унции
- [ ] Температура: °C, °F, K
- [ ] Объём: мл, л, галлоны (US), чашки (US)
- [ ] Конвертация происходит в реальном времени при вводе (без кнопки "Конвертировать")
- [ ] Кнопка swap меняет местами исходную и целевую единицу, пересчитывает мгновенно
- [ ] Режим "все единицы": показать результат во всех единицах выбранной категории одновременно
- [ ] Тап по любому результату копирует значение в буфер обмена с визуальным подтверждением
- [ ] Точность: до 6 значащих цифр, trailing zeros убираются
- [ ] Температура конвертируется по формулам (не коэффициентами)
- [ ] Ввод допускает десятичные дроби и отрицательные числа (температура)

### UI / UX

- [ ] Single HTML file, zero dependencies, inline CSS + JS
- [ ] Dark theme в стиле iOS (фон #1C1C1E, карточки #2C2C2E, акцент #007AFF)
- [ ] Mobile-first, контейнер max-width 400px, центрирован
- [ ] Табы категорий — горизонтальная полоска сверху, активный таб подсвечен
- [ ] Поле ввода: крупный шрифт (min 24px), type="number", autofocus при открытии
- [ ] Выбор единиц — dropdown или pill-селектор, достаточно крупный для пальца (min 44px tap target)
- [ ] Кнопка swap между селекторами единиц: иконка стрелок, анимация поворота при нажатии
- [ ] Список "все единицы" — компактные карточки с названием единицы и значением
- [ ] Haptic-подобная визуальная обратная связь при копировании (brief flash / чекмарк)
- [ ] Плавные transitions при смене категории (opacity/transform, 200ms)
- [ ] Нет скроллов на основном экране при 4-5 единицах; при показе всех единиц — вертикальный скролл допустим

### Производительность

- [ ] Файл весит менее 30 КБ
- [ ] Конвертация без задержки (синхронный расчёт, без debounce)
- [ ] Работает полностью оффлайн

### Доступность

- [ ] Навигация клавиатурой (Tab, Enter)
- [ ] Поле ввода получает фокус при загрузке
- [ ] Контраст текста соответствует WCAG AA (4.5:1)

## Приоритеты

### P0 (MVP) — этот релиз

- Все 4 категории с полным набором единиц
- Real-time конвертация при вводе
- Режим пары (из A в B) + swap
- Режим "все единицы категории"
- Копирование результата тапом
- iOS dark theme, mobile-first
- Клавиатурная доступность

### P1 (v2)

- История последних 10 конвертаций (localStorage)
- Избранные пары (pin)
- Дополнительные категории: площадь, скорость, время, давление
- Поиск единицы по названию
- Поддержка ввода через запятую (1,5 = 1.5)

### P2 (будущее)

- Валюты (с внешним API курсов)
- PWA: установка на домашний экран, иконка
- Голосовой ввод ("сто фунтов в килограммы")
- Шаринг результата (Web Share API)

---

## Ограничения

- Single HTML file, без внешних зависимостей (no CDN, no frameworks)
- Никакого серверного кода — всё на клиенте
- Все формулы конвертации захардкожены (кроме будущих валют)
- US галлоны и US чашки (не Imperial)
- Не поддерживаем научную нотацию в MVP
